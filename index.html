SublimeText3 Node插件配置方法

发表于 2016-08-13   |   分类于 Nodejs

Step1:

从githb上下载nodejs的插件，并把文件夹名字改为Nodejs
https://github.com/tanepiper/SublimeText-Nodejs

Step2:

打开Sublime Text3，点击菜单“Perferences” =>“Browse Packages”打开“Packages”文件夹，并将第1部的Nodejs文件夹剪切进来

Step3:

打开文件“Nodejs.sublime-build”，将代码 “encoding”: “cp1252” 改为 “encoding”: “utf8” ，将代码 “cmd”: [“taskkill /F /IM node.exe & node”, “$file”] 改为 “cmd”: [“node”, “$file”] ，保存文件

Step4:

打开文件“Nodejs.sublime-settings”，将代码 “node_command”: false改为 “node_command”: “D:\Program Files\nodejs\node.exe” ，将代码 “npm_command”: false 改为 “npm_command”: “D:\Program Files\nodejs\npm.cmd” ，保存文件

Step5:

自己在SublimeText里写一个js的demo，按ctrl+B测试一下。

其中 “D:\Program Files\nodejs\node.exe和D:\Program Files\nodejs\npm.cmd”为你本地nodejs安装目录。我的是在C盘下

关于java.util.ConcurrentModificationException原因

发表于 2016-08-13   |   分类于 Java

前言

今天在添加处理图片集合的时候，出现了这个错误。翻译过来的意思是，并发修改的异常。于是，看来一下自己的代码。我在把图片传给服务器之后，把图片的信息保存在一个List集合里，在删除的时候，将图片的信息从这个集合里删除掉，添加图片和删除图片都用的异常网络请求，这样就必然会导致ConcurrentModificationException。

错误的代码

原因1:

由于自己代码的规范问题multiUploadPhoto(List imageItemList)的参数，刚好写的imagePicker。

原因2:

for(ImageItem imageItem:imageItem)其原理是使用Iterator来实现的，当我在把图片信息保存在imageItemList里的时候，集合的size发生了变化

解决方法:

在增加或删除的时候，把要增加或要删除的元素放在一个临时数组里
